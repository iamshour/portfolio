Let's keep in touch, yeah?

Project

LetsTalk
Mobile Chatting app

Genre

UI/UX

Year

2020

Org

Side Project

Situation

I started this side project during my free time in the early months of 2020, during Covid-19’s lockdown. My plan was to clone whatsapp’s UI and add my inspiration to produce a simple easy and interactive mobile chat app. I spent a week on preparing sketches and low-fidelity wireframes, before transforming the mockups into interactive UI screens using Figma, then prototyping UI screens and mirroring final prototypes using Figma Mirror app on my android device for testing and screen recording.

Preparation

Sketching, Color Palette, Logo, Font & Tools used

The very first steps of this project was sketching breif UI elements on my notebook and creating low-fidelity wireframes. This allowed me to conclude a very basic look and feel of the app’s UI. Next, it was time to pick the right color Palette to work with. I picked “Teal Blue” as the primary color, with two lighter tints, and light usage of “Green Teal” and regular white.
Then my attention turned towards designing a simple logo related to chats and messaging, the end product consisted of two chat bubbles, each with a tint of the primary color, and one contained the app’s name “LetsTalk”.
My choice for the primary font was Google’s Sans-Serif font: “poppins”, with a lighter usage of a cursive font: “Righteous”.
In addition to using a selection of Google’s material design icons as well as some of undraw’s illustrations throughout my project.

Action

My Goal is to essetially turn sketches/ideas into engaging user interface, using previously mentioned design resources and tools.
Steps are illustrated below:

Login, Verfication, and account Setup

The first phase in using LetsTalk app would be signing in/signing up, verifying the phone number, and setting up the account. I worked on designing on each step individually, in a simultaneous process, by which every step starts after users complete the one before. I’ll illustrate a little on each step below.

First Step: Sign in / Sign up

The first user interaction with LetsTalk is signing in with a previously created account, or signing up for a new account. Both steps could be done using three methods:

  1. Through a user’s Google account
  2. Through a user’s Facebook account
  3. Through a user’s phone number directly. All three methods require including the phone number for Verification.

Second Step: Verification

The second is verifying your phone number after recieving a text message containg OTP code. A simple and familiar design for users, with four input box containers for the OTP code, each with a 56px width and height, and 16px of border-radius around all corners. An option to send a new code is also included, with the “Verify” botton just above the keyboard.

Third Step: Account Setup

The third and last step for new users is to setup their account, by filling up a brief bio, uploading a profile picture, and adding user info such as name, birthday, and email address. Each input container includes a placeholder text, and an icon on the right side to clear everying up. Users can press on the “Create Account” botton when they’re done and ready to use the app.

Main App UI

Users can navigate between three main tabs, the first one is the chatting tab, where previous and new chats are displayed in a chronological order, the second tab is for Call Logs, which stores all previous calls with a call icon on the right side of each call log, and the third tab is about adding new friends, which displays a suggestion list based on mutual friends, and a search icon to look for specific friends to add.

Chats between users

The main goal of the app is obviously for users to chat between each other. I desinged two chat containers with different colors, where chats sent from users are contained in a “Green Teal” colored bubble, and chats recieved by users are contained in a light tint of grey bubble. Users can exhange regular texts, emojis, pictures, voice notes, or other attached media. Previous calls or missed calls are also displayed among a chat. I also designed another screen UI, which is displayed after a user presses on the piucture/name of the person he’s chatting with. It displays an option to Turn notifications on/off, and displays previously exchanged media, bookmarks, or events with this person.

Chats within a Group

Another important feature is to create a group and exhange chats, media, or events within group members. Users can even press on the Group’s picture/name to view the group’s previously exchanged media, or mute the group’s notification. A full list of group members is also displayed in the bottom, with a search icon to find a group member easier and faster.

Making a Call

A mandatory feature of almost every chatting app, where users can exchange calls relying on WI-FI instead of cellular plans. Users can put a call on speaker or on mute, and after a call ends, its duration is directly displayed.

Final Thoughts

Even though this was a side project and not for a specific firm/organization, it was quite an important way to apply my UI skills and inspiration in a full mobile app UI design. Although i didn’t to go through any research or survey, I rellied on my background using different chatting platforms, to support and inspire my last brief skteches and wireframes, which i transformed in about 5-10 days into an interactive mobile interface.
